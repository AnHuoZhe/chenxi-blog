一、项目背景

晨汐博客是一个个人AI学习知识花园，使用 Eleventy + Nunjucks 构建，部署在 GitHub Pages (https://anhuozhe.github.io/chenxi-blog/)。

核心隐喻是生物学：细胞（最小知识单位）→ 器官（按前端/后端/部署/AI/数据五大类组织的技能模块）→ 个体（完整项目复盘）。外加一个组织wiki区（术语自动链接）。

写作风格：最少的话 + 最活的比喻。目的是让人"有这个概念的印象，能看懂AI的输出"。

现在功能完整但UI太简陋——只有基础CSS，没有视觉层次，没有把生物隐喻的"知识花园"感觉做出来。

二、开始前先读取以下文件

在动手改任何代码之前，先读这些文件了解现状：
- _includes/layouts/base.njk —— 全局HTML骨架
- _includes/layouts/article.njk —— 文章详情页布局
- index.njk —— 首页模板
- 细胞.njk、器官.njk、个体.njk —— 三个分类页模板
- 404.njk —— 404页面
- styles.css —— 当前所有样式（约2.4KB）
- eleventy.config.mjs —— Eleventy构建配置（只读，了解集合和过滤器结构）
- src/terminology.mjs —— 术语自动链接逻辑（只读，了解a标签生成方式）

- 构建工具：Eleventy + Nunjucks

四、设计参考方向

参考以下设计风格，取它们的优点融合：

1. overreacted.io —— 极简排版，专注阅读体验，干净的字体层级。这是简洁度的基准线。
2. Maggie Appleton 的数字花园 —— 只取两个概念："有机感"（温暖的配色，内容像在生长）和"内容有生长阶段标签"。不照搬其动画、插画、复杂交互系统。
3. Notion风格的文档站点 —— 侧边栏+内容区的清晰信息架构参考。

实现层面的底线：以 overreacted.io 的简洁度为基准，往上加一点花园感。不要做成重前端应用。

五、具体要改什么

A. 全局改动

1. 字体栈升级：中文正文用 Noto Sans SC / 思源黑体等无衬线字体，代码用 Fira Code 或 JetBrains Mono 等宽字体
2. 暗色模式：用 CSS 变量 + data-theme 属性实现 light/dark 双模式。暗色背景用深墨绿系（如 #1a221c 左右），保持绿色主调的识别度，不要纯黑背景。默认跟随系统 prefers-color-scheme，header 加一个手动切换按钮（原生JS，不超过30行）
3. 响应式打磨：手机端导航改为汉堡菜单（原生JS），卡片单列排布，字号适当缩小
4. 过渡动画：链接 hover、卡片 hover、暗色切换加轻微 transition（0.2s左右），不要太花
5. Footer 丰富一点：加 GitHub 链接、RSS 链接（如果有的话）、"持续生长中"的文案。简单干净即可

B. 首页

当前首页：头像+名字+简介 + "最近更新"卡片列表。太像普通博客首页。

改为"知识花园入口"风格：

- 顶部保留头像和名字，旁边一句话改成更有花园感（如"种一棵 AI 知识树，从看懂一个细胞开始"）

- 下方加四个花园入口卡片（横排引导进入四个区）：
  桌面端一行四个 | 平板端 2x2 网格 | 手机端垂直堆叠
  每个卡片：emoji图标 + 区名 + 一句话描述，点击进入对应页面
  - 🌱 细胞 "从最小概念开始，把术语看懂" → /细胞/
  - 🌿 器官 "按用途学习，五大分类" → /器官/
  - 🌳 个体 "完整项目，从零到一" → /个体/
  - 📖 组织wiki "术语词典，自动链接" → /组织wiki/

- 再往下是"最近更新"的卡片列表，保留当前结构

C. 分类页（细胞/器官/个体）

- 细胞页：顶部有该区解释文字+🌱图标（不用emoji的话用CSS画的简单图形也可以），下方卡片列表加hover微交互
- 器官页：当前按五大类分组（前端/后端/部署/AI/数据），保持这个结构。分组之间视觉分隔更明显，每组加分类图标。五大类分组标题不要动——它们来自 eleventy.config.mjs 的 organGroups 集合
- 个体页：保持当前结构，项目状态标签（进行中/已完成等）加视觉区分

D. 文章详情页

- 面包屑导航更明显，当前用 › 分隔符，样式上让它更可点、更可读
- 标题下方加元信息行：日期 + 所属区域标签 + 阅读时间估算（纯CSS/模板计算，不需要JS）
- 正文排版优化：行高1.8以上，段落间距舒适（margin-bottom: 1.2em左右），代码块有独立的视觉样式（浅灰/墨绿背景+左侧绿色细色条）
- 术语自动链接视觉提示：
  CSS能做的：自动链接用虚线下划线（text-decoration-style: dotted）+ 淡绿色 + hover变色
  如果需要hover显示术语简短定义：需要微调 src/terminology.mjs 的 linkTerminology 函数，在生成的 a 标签上加 title 属性。此项标记为"简单就做，复杂就跳过"——CSS视觉提示已经够用了
- 文章底部加"← 返回XX区"的导航链接

E. 404页面

保持当前结构，视觉好看即可，不用花精力。

六、约束

- 主战场是 styles.css，尽量只改这一个文件
- 可以微调 Nunjucks 模板（加class、加结构、加link标签），但不动模板里的Nunjucks逻辑（集合循环、条件判断、autoLinks过滤器调用等）
- eleventy.config.mjs 完全不碰
- 不引入JS框架（React/Vue/jQuery），暗色切换和汉堡菜单用原生JS，总共不超过60行
- 如果暗色模式可以通过CSS变量+data-theme属性实现，就不需要单独的dark-mode.css
- 保持构建产物体积小，不引入大型CSS框架（Bootstrap/Tailwind等）
- 颜色系统保持绿色主调，暗色模式用深墨绿系
- 路径前缀 /chenxi-blog/ 不能丢，所有链接用 {{ '...' | url }} 过滤器
- 不创建git worktree，直接在当前项目目录操作

七、产出标准

以下三项满足即可交付，细节打磨留到使用中按需迭代：

1. 首页有四个区入口卡片 + 最近更新列表，视觉层次清晰
2. 文章详情页可读性明显提升（行高、段落间距、代码块样式）
3. 暗色模式可切换，颜色不违和
